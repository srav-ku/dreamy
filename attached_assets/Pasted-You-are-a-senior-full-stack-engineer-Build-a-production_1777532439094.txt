You are a senior full-stack engineer. Build a production-ready image gallery platform with a separate admin portal. Follow the instructions strictly and implement clean, scalable architecture.

Do NOT overcomplicate. Avoid unnecessary libraries. Focus on performance, clean UI, and maintainability.

---

# 1. PROJECT STRUCTURE

Create a monorepo:

/album
├── /backend-workers      # Cloudflare Workers API (Hono)
├── /public-site          # Public gallery (React + Vite)
└── /admin-portal         # Admin dashboard (React + Vite)

---

# 2. TECH STACK (STRICT)

Frontend:

* React (Vite)
* Tailwind CSS
* React Router
* Fetch API (no heavy libs unless necessary)

Backend:

* Cloudflare Workers
* Hono framework

Database:

* Cloudflare D1 (SQLite)

Image Storage:

* Cloudinary (direct upload from frontend)

---

# 3. DATABASE SCHEMA (D1)

Create tables:

## persons

* id (primary key)
* name (text)
* slug (unique, indexed)
* profile_image (text)
* bio (text)
* created_at (timestamp)
* updated_at (timestamp)

## albums

* id
* person_id (foreign key)
* name
* slug

## images

* id
* album_id (foreign key)
* image_url (text)
* thumbnail_url (text)
* order_index (integer)
* created_at (timestamp)

---

# 4. BACKEND (Hono + Workers)

Implement REST API.

## Public endpoints:

* GET /api/persons
* GET /api/person/:slug
* GET /api/person/:slug/full   ← IMPORTANT (optimized response)
* GET /api/albums/:personSlug
* GET /api/images/:albumSlug
* GET /api/search?q=

Search:

* Convert names to lowercase
* Use LIKE 'query%' for better performance

---

## Admin endpoints:

* POST /api/admin/login
* POST /api/admin/person
* PUT /api/admin/person/:id
* POST /api/admin/album
* POST /api/admin/images

---

# 5. ADMIN SECURITY (FIXED)

DO NOT use raw password in every request.

Implement:

* POST /api/admin/login

  * Accept password
  * If valid → return a simple token (from env)

* Store token in frontend (localStorage)

* All admin routes:

  * Require Authorization header with token
  * Validate against env variable

---

# 6. CLOUDINARY INTEGRATION (IMPORTANT)

Upload images directly from frontend (admin portal).

Flow:

1. Upload image → Cloudinary
2. Get URL
3. Send URL to backend
4. Store in DB

DO NOT upload images through Workers.

---

## Image optimization (MANDATORY):

* Use thumbnails in grid:

  * width ~300px
  * auto format + compression

* Use full image only in viewer
