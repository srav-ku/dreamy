You are a senior full-stack engineer. Build a production-ready, minimal, high-quality image gallery platform with a separate admin portal.

Do NOT overcomplicate. Focus on clean architecture, scalability, and modern UI similar to premium platforms (inspired by X/Twitter style: clean, centered, minimal, smooth).

---

# 1. PROJECT OVERVIEW

Create TWO separate apps:

1. Public Website (for users)
2. Admin Portal (completely separate app, not included inside main site UI)

Both should share the same backend (Cloudflare Workers API + Cloudflare D1 database + Cloudinary for images).

No authentication for users.
Admin portal can use simple password protection (env-based).

No chat feature.

---

# 2. TECH STACK (STRICT)

Frontend:

* React (with Vite)
* Tailwind CSS
* React Router
* Axios or Fetch API

Hosting:

* Cloudflare Pages

Backend:

* Cloudflare Workers (API layer)

Database:

* Cloudflare D1 (SQLite)

Image Storage:

* Cloudinary

---

# 3. DATABASE DESIGN (Cloudflare D1)

Create tables:

### persons

* id (primary key)
* name (text)
* slug (unique)
* profile_image (text)
* bio (text)
* created_at (timestamp)

### albums

* id
* person_id (foreign key)
* name
* slug

### images

* id
* album_id (foreign key)
* image_url (text)
* thumbnail_url (text)
* created_at

---

# 4. BACKEND (Cloudflare Workers API)

Create REST API endpoints:

### Public:

* GET /api/persons
* GET /api/person/:slug
* GET /api/albums/:personSlug
* GET /api/images/:albumSlug
* GET /api/search?q=

### Admin:

* POST /api/admin/person
* PUT /api/admin/person/:id
* POST /api/admin/album
* POST /api/admin/images

Use JSON responses only.
